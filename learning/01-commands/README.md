# 01. Commands 模块

## 概述

Commands（斜杠命令）是 Claude Code 中最直接的交互方式，通过简单的 `/命令名` 即可触发预定义的工作流程。

## 核心命令详解

### 1. `/plan` - 功能实现规划

**用途**: 在实现新功能前，先制定详细的实现计划

**使用场景**:
- 添加新功能
- 重构大型模块
- 多文件修改
- 不清楚具体实现方案时

**示例**:
```
/plan
```
然后告诉 Claude 你想做什么，比如："添加用户登录功能"

---

### 2. `/tdd` - 测试驱动开发

**用途**: 强制按照 TDD 流程开发：先写测试 → 实现功能 → 重构

**工作流程**:
1. 定义接口
2. 编写失败的测试 (RED)
3. 实现最小代码让测试通过 (GREEN)
4. 重构优化 (REFACTOR)
5. 确保测试覆盖率 ≥80%

---

### 3. `/e2e` - 端到端测试

**用途**: 自动生成 Playwright E2E 测试

**功能**:
- 创建测试旅程
- 运行测试
- 捕获截图/视频/追踪
- 上传测试产物

---

### 4. `/code-review` - 代码审查

**用途**: 对代码进行全面的质量和安全审查

**检查项**:
- 代码质量
- 安全漏洞
- 可维护性
- 最佳实践
